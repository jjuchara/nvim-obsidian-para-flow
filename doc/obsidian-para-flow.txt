INTRODUCTION                                            *obsidian-para-flow*

obsidian-para-flow provides a Neovim workflow for capturing notes into an
Obsidian Inbox and processing them into a configurable PARA structure through
the official Obsidian CLI.

The current implementation includes configuration, health diagnostics,
terminal-first Inbox note creation through QuickAdd, and FIFO Inbox review.
Review opens the oldest note as an editable Markdown buffer with persistent
status and action regions. The p, a, r, x, d, e, s, and q actions are active.
PARA sorting completes all pickers and preflight checks before mutation, adds
missing metadata, moves last, and rolls properties back on failure. Exact
target-name conflicts open a side-by-side resolver with rename, delete, and
editable merge-preview flows.

===============================================================================
REQUIREMENTS                               *obsidian-para-flow-requirements*

- Neovim 0.10 or newer.
- Obsidian installed with the 1.12.7 or newer installer.
- Obsidian CLI enabled.
- QuickAdd 2.12 or newer with a configured Inbox choice. Its filename format
  and template must use the named {{VALUE:title}} variable. The plugin also
  supplies QuickAdd's reserved value input for Template-choice compatibility.

There are no mandatory Neovim runtime dependencies.

===============================================================================
INSTALLATION                               *obsidian-para-flow-installation*

Install with the plugin manager of your choice. For lazy.nvim:
>
  {
    "jjuchara/nvim-obsidian-para-flow",
    version = "*",
    opts = {
      -- Add the complete mandatory configuration from the Setup section.
    },
  }
<
The version range follows the newest semantic-version release. Run :Lazy update
after a release to upgrade the plugin and refresh lazy-lock.json. Remove version
only when intentionally following every commit on main.

The plugin is Lua-only and has no build step. Run |:checkhealth| after setup and
use |:ObsidianParaHealth| to verify the external Obsidian requirements.

===============================================================================
SETUP                                                 *obsidian-para-flow-setup*

All vault-specific paths are mandatory:
>
  require("obsidian-para-flow").setup({
    vault = "My Vault",
    inbox = {
      folder = "6. Inbox",
      quickadd_choice = "inbox",
    },
    para = {
      projects = { folder = "1. Projects", link = "[[My Projects]]" },
      areas = { folder = "2. Areas", link = "[[My Areas]]" },
      resources = { folder = "3. Resources" },
      archives = { folder = "4. Archives" },
    },
  })
<
Optional defaults:
>
  mappings = {
    new = "<leader>on",
    review = "<leader>oi",
  },
  review = {
    layout = "float",
    width = 0.7,
    height = 0.7,
    winblend = 0,
  }
<
A mapping can be set to false to disable it. Float dimensions below 1 are
screen fractions; values of 1 or more must be whole columns or rows.
The default float occupies 70% of the available width and height and uses a
centered titled frame, inset Markdown body, compact
queue line, persistent bracketed command bar, and the active theme's Pmenu
popup surface. winblend accepts a whole number from 0 (opaque) to 100 and
defaults to 0 so lower-buffer text cannot bleed through.

If Obsidian is not running, the first command opens the configured vault,
waits up to 15 seconds for CLI readiness, and retries the command once. Inbox
creation verifies the exact vault name before running QuickAdd.

===============================================================================
COMMANDS                                           *obsidian-para-flow-commands*

                                                *:ObsidianParaInboxNew*
:ObsidianParaInboxNew
    Prompt for a title in Neovim, run the configured QuickAdd choice without
    Obsidian UI, identify the single new Inbox Markdown file, open it in the
    current window, and place the cursor at the body. If QuickAdd leaves an
    exact <% tp.file.cursor() %> marker, remove it from the buffer and place
    the cursor there; otherwise use the position after frontmatter and the
    first H1.

                                             *:ObsidianParaInboxReview*
:ObsidianParaInboxReview
    Load the FIFO Inbox queue and open its oldest note in the configured float
    or fullscreen layout. The status shows queue position and note path. The
    footer keeps p/a/r/x/d/e/s/q visible. d saves, asks for confirmation with
    Cancel first, and moves the note through the Obsidian trash operation only.
    It advances after CLI success and otherwise leaves the current note open.
    e saves, pauses review, and opens the
    note in the original window. s saves and skips the note for this pass. q
    exits, prompting to cancel, save, or discard when the buffer is modified.
    Saving actions detect external file changes and write failures before
    changing session state. p/a/r/x collect their required input, apply missing
    metadata transactionally, and move the note last.

    An exact destination conflict opens labeled read-only target and Inbox
    panes. <Tab> switches panes; m opens Merge Preview, r chooses a new final
    name, d trashes the Inbox source after confirmation, and q returns to
    review. Merge Preview is editable and exposes local <leader>om to commit
    and <leader>oq to return. Commit revalidates both source snapshots, writes
    the target, trashes the Inbox source last, and restores the target on
    failure. An incomplete restore halts review with recovery details.

                                                   *:ObsidianParaHealth*
:ObsidianParaHealth
    Run read-only checks for Neovim, Obsidian CLI, the configured vault,
    QuickAdd choice, and PARA folders.

===============================================================================
REVIEW ACTIONS                                  *obsidian-para-flow-actions*

p    Move to Projects. Select a folder and, when missing, an #area note.
a    Move to Areas. Select the configured root or a nested folder.
r    Move to Resources. Select a folder and, when missing, an #area note.
x    Move to Archives. Select a folder and enter a missing archive reason.
d    Save, confirm, and move the Inbox note to the Obsidian trash.
e    Save, pause review, and edit the note in the originating window.
s    Save, skip this note for the current pass, and advance.
q    Quit; modified notes require explicit cancel, save, or discard.

All sorting prompts and preflight checks finish before mutation. Properties
are applied before the final move and rolled back when the move fails. An
external file change stops a saving action without advancing the queue.

During an exact destination conflict, <Tab> switches the target and Inbox
panes. m opens Merge Preview, r selects another final name, d confirms trash,
and q returns to review. In Merge Preview, <leader>om applies the merge and
<leader>oq returns without committing it.

===============================================================================
MAPPINGS                                           *obsidian-para-flow-mappings*

<leader>on    Create a new Inbox note.
<leader>oi    Start Inbox review.

Both mappings include descriptions and can be changed or disabled in setup().
When WhichKey is available, the default <leader>o prefix is labeled
"obsidian para flow" and shown with a purple crystal icon. WhichKey remains
optional.

The plugin calls vim.ui.input() and vim.ui.select() at interaction time. Stock
vim.ui and providers such as Snacks therefore share the same plugin contract;
no provider is a dependency.

TROUBLESHOOTING                         *obsidian-para-flow-troubleshooting*

"setup() must be called first" ~
    Configure the plugin before invoking its Lua API or commands.

"Obsidian CLI" spawn or unavailable errors ~
    Install Obsidian 1.12.7 or newer with the current installer, enable its
    CLI, and keep Obsidian running. The plugin attempts one bounded startup.

Unexpected vault name ~
    The setup vault must exactly equal the name open in Obsidian. The plugin
    fails closed before QuickAdd when another vault is resolved.

QuickAdd check or capture failure ~
    Install QuickAdd 2.12 or newer. Confirm the configured choice exists and
    that both its filename and template use {{VALUE:title}}. The plugin passes
    the title as both value-title and value-value for QuickAdd 2.12.3 Template
    choices. Existing or unsafe Inbox filenames are rejected before QuickAdd
    runs.

Review reports an external change ~
    The note changed after review loaded it. Preserve both versions, reconcile
    them manually, and start a new review pass. The blocked action does not
    advance the queue.

Review is halted with recovery details ~
    A transaction could not complete its rollback. Do not repeat mutations.
    Use the reported source, destination, properties, and compensation errors
    to restore the disposable or backed-up vault before restarting review.

DEVELOPMENT                                     *obsidian-para-flow-development*

The project has no build step.

  make test       Run isolated mini.test tests.
  make lint       Run Selene, StyLua checks, shell syntax, and launcher tests.
  make format     Format Lua files with StyLua.
  make check      Run the complete local automated gate.
  make helptags   Regenerate doc/tags after editing this help file.

The opt-in disposable-vault gate is:
>
  make test-integration TEST_VAULT=<name>
<
It verifies the exact vault and runs a non-overwriting lifecycle on a uniquely
named, frontmatter-marked fixture before sending it to the vault trash. Never
use a production vault. RELEASE_CHECKLIST.md records the manual, provider,
rollback, CI, and end-to-end evidence required before a stable tag.
MANUAL_TESTING.md contains the exact fixture and fault-injection procedure.

Use ./scripts/nvim-dev for an isolated LazyVim profile and fixture vault. See
CONTRIBUTING.md in the repository for first-time vault registration and reset
instructions.
